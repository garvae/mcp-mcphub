# Code of Conduct

## Our Standard

Contributors are expected to keep technical discussion direct, respectful, and evidence-based.

The project welcomes contributors with different levels of MCP, MCPHub, and open-source experience as long as they engage constructively and in good faith.

Examples of expected behavior:

- discuss behavior and tradeoffs concretely
- report bugs with enough detail to reproduce them
- accept correction when code, docs, or assumptions are wrong
- keep security-sensitive details private until there is a safe disclosure path

Examples of unacceptable behavior:

- harassment or personal attacks
- intentionally misleading technical claims
- publishing secrets, exploit details, or private infrastructure information without need
- hostile or exclusionary language

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, or pull requests that violate these standards.

For serious conduct concerns, contact the repository owner privately through GitHub.
