# Roadmap

This roadmap is directional, not a promise.

## Near-term

- improve client setup examples across real MCP clients
- track upstream MCPHub API drift faster
- expand compatibility and packaging tests
- improve generated tool descriptions and risk notes

## Ongoing

- keep safe defaults and visible risk boundaries
- improve docs based on real user reports
- add coverage for new MCPHub management routes
- tighten diagnostics and troubleshooting guidance

## Not Planned

- replacing MCPHub itself
- acting as a generic arbitrary HTTP proxy
- replacing downstream `/mcp` passthrough behavior
- publishing an official Docker image unless that decision is revisited explicitly
