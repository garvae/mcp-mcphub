## Automated Matrix

- `pnpm test:compatibility` runs the real-container smoke suite in `tests/integration/upstream-compat.test.ts`.
- Set `RUN_MCPHUB_COMPAT_TESTS=1` to enable the suite locally.
- Optionally scope the suite with `MCPHUB_COMPAT_IMAGE_TAG=<tag>`.
- GitHub Actions runs the compatibility matrix on `1.0.15`, `1.0.20`, `latest`.

## MCP Client Compatibility

| Client era                      | Recommended server mode                         | Notes                                                                                                                   |
| ------------------------------- | ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Legacy `2025-11-25` and earlier | `MCP_HTTP_MODE=stateful`                        | Works with the SDK range declared in `package.json` (`^1.29.0`)                                                         |
| Modern `2026-07-28`             | `MCP_HTTP_MODE=stateless`                       | Use raw JSON-RPC over HTTP today; header/body validation is enforced server-side                                        |
| Dual-era deployments            | separate listeners or a deliberate rollout plan | The repository keeps both modes available behind one env flag rather than auto-negotiating eras inside one process path |
